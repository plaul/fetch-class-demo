fetch with local caching of data

This example uses (will eventually) the following strategy to interact with the backend

It fetches data from the server. STORES the data locally for future use (filter, sort, etc.) and build the table-rows from these data.

For ADD, PUT and DELETE operations it delegate these operations to the server, but, it also updates the local representation of data, and re-renders the page from this updated list as outlined below

  • POST → For a successful POST operation it adds the new user (now with ID) returned by the server to the local list, and re-renders the table using this list
  • PUT → For a successful PUT operation it adds the edited user returned by the server to the local list, and re-renders the table using this list
  • DELETE → For a successful DELETE operation it removes the deleted user from the local list AFTER the server has responded with OK, and re-renders the table using this list

Eventually we will add functionality to automatically update the list with potential changes from the server
